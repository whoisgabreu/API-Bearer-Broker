Autenticação

Para realizar solicitações nesta API é necessário enviar a sua API Key no cabeçalho X-API-KEY.

# API_KEY
ccf9d74a30dc58035c50d1d0cb19dd20

# Cabeçalho HTTP
X-API-KEY: ccf9d74a30dc58035c50d1d0cb19dd20
⚠️ Importante: mantenha sua API Key em segredo. Evite comitar chaves em repositórios públicos e prefira variáveis de ambiente.

Exemplo (cURL)

curl -X GET   "https://api.v4lisboatech.com.br/analise/lead?socio=Fulano&empresa=Minha%20Empresa"   -H "X-API-KEY: ccf9d74a30dc58035c50d1d0cb19dd20"

Exemplo (JavaScript fetch)

const headers = {
  "X-API-KEY": "ccf9d74a30dc58035c50d1d0cb19dd20"
};
GET /analise/lead

O que faz? Procura informações sobre o sócio e a empresa informados e verifica a presença online.

Parâmetros de consulta (query)

socios: string   # nome_do_socio
empresa: string # nome_da_empresa

Requisição

# Forma 1 — com objeto de parâmetros
GET https://api.v4lisboatech.com.br/analise/lead?socio=<nome_do_socio>&empresa=<nome_da_empresa>

Resposta

HTTP/1.1 200 OK
Content-Type: application/json

{
  "socio": "Fulano de Tal",
  "empresa": "Minha Empresa LTDA",
  "presenca_online": {
    "google_ads": true,
    "meta_ads": false,
    "google_business": true
  },
  "fontes": [ ... ]
}
Dica: utilize encodeURIComponent ao montar a URL para nomes com espaços e acentos.
GET /broker/extrair_bearer

O que faz? Coleta o Bearer de autenticação da página do V4 Broker.

Requisição

GET https://api.v4lisboatech.com.br/broker/extrair_bearer

Resposta

HTTP/1.1 200 OK
Content-Type: application/json

{
  "bearer": "Bearer xxxxxx.yyyyyy.zzzzzz"
}
POST /convert/docx

O que faz? Converte um arquivo .docx em .pdf.

Corpo (JSON)

{
  "file_base64": "<string_binária_do_DOCX>"
}

Requisição

POST https://api.v4lisboatech.com.br/convert/docx
Content-Type: application/json
X-API-KEY: <SUA_API_KEY>

{ "file_base64": "<string_binária_do_DOCX>" }

Resposta

HTTP/1.1 200 OK
Content-Type: application/json

{
  "pdf_base64": "<string_base64_do_pdf>"
}
